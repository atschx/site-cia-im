## 快速开始

这种文章发出来就是浪费大家时间吗？如果某乎的算法足够好，把它推荐给一个 React 的初学者，又喜欢自己亲自动手，给他埋上一个知识的种子，是不是有用呢 ？

有用，但价值有限。信息没有经过加工，没有润色上作者的实践心得及看法。每一个在互联网上发生的人， 都应该对自己的言行负责，不做匿名的英雄，而是实名的勇士。

我最喜欢对npm install的过程吐槽，仿佛这个环节就是在浪费生命，可是没有它还不行。所以接受吧，这就是宿命。这个环节大量的网络 IO 和 本地文件 IO，这个没办法语言的包管理机制所致，除非你不写前端。

有人可能会杠，你用 Go 的包管理时不是这样吗 ？你用 Java 的 Maven 时不也这样吗 ？矫情啥，嗯。你说的对。只是这个 install 每个碎文件的大小小到令人发指，甚至几行代码就是一个 package。
